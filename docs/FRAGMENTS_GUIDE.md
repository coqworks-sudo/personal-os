# Fragments 運用ガイド

**Fragments** は、けんたろうとClaudeのやり取りから生まれた思考の断片・気づき・調査依頼を蓄積する外部記憶システムです。Slackの2チャンネルで構成され、どのClaudeとの会話スレッドからでも読み書きできます。

---

## チャンネル構成

### #research-queue
**用途：** 調べてほしいこと・確認してほしいことの投入口

ひらめいた疑問や調査依頼をここに投げる。Claudeが意図を確認してから自律調査→レポート保管する非同期リサーチキュー。本筋の作業を止めずにひらめきを「預ける」場所。

**投稿フォーマット（自由書きでOK）**
- 何を知りたいか
- なぜ知りたいか（任意）
- どの粒度で知りたいか（「概要だけ」「実装レベルまで」など）
- 末尾に `[調査希望]` または `[メモ]` を添えると判別しやすい

**処理フロー**
1. けんたろうがこのチャンネルに投稿
2. Claudeが「こういう理解でOK？」と意図確認（スレッドで返信）
3. OK → 自律調査開始
4. Markdownレポート保存 + Slackに完了通知

**Claudeの処理確認ルール**
Claudeが #research-queue の投稿を処理した場合、回答をどこに返すかに関わらず、
必ず元の投稿にスレッド返信を行う（「受け取りました」一言でも可）。
- スレッドになっている = 処理済み
- スレッドなし = 未処理

この状態管理により、けんたろうがキューを確認する際にスレッドの有無だけで処理状況を判断できる。

**自動化ステータス**
- Phase 1 手動運用（投稿→Claudeに依頼）：運用中
- Phase 2 n8nでSlack → Claude Code自動トリガー：未着手
- Phase 3 ハートビートによる完全自律：未着手

### #discoveries
**用途：** 気づき・感動・アイデア・思考の断片の記録

ふとした気づき、会話からのインスピレーション、後から深掘りしたいアイデアの種をここに書き留める。調査が必要なものは #research-queue へ。

**こんなことを書く**
- 新しいツールや仕組みを試して感動したこと
- 「これ使えるかも」というひらめき
- プロジェクトの方向性に関わる気づき
- 将来深掘りしたいアイデアの種

**活用例（Claudeへの依頼）**
- 「先月の気づきをまとめて」
- 「事業設計に関係しそうな発見を抽出して」
- 「BRIDGE.mdに盛り込むべきInsightを探して」

---

## 投稿の判断基準

| 内容 | 投稿先 |
|---|---|
| 調べてほしいこと（疑問・調査依頼） | #research-queue |
| 気づき・感想・ひらめきメモ | #discoveries |
| 曖昧なもの | どちらでもOK（Claudeが内容から判断） |

迷ったときは自由に投げてOK。末尾に `[調査希望]` か `[メモ]` を添えると確実。

## レビューサイクル

隔週土曜 9:00〜10:00 に定期レビューセッションを設定済み。

- #discoveries の蓄積を棚卸し
- 気づきをBRIDGE.mdに蒸留
- 調査済みトピックを深掘り・実行タスクへ変換

---

## Claudeへの共有方法

このファイルをプロジェクトの前提ファイルとして配置することで、どのスレッドのClaudeも初期状態からFragmentsの存在と使い方を把握できます。

新スレッドでの最初の指示例：
```
FRAGMENTS_GUIDE.mdを読んでFragmentsの仕組みを把握した上で会話を始めてください。
```
